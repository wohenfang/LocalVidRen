# LocalVidRen - 本地短视频智能重命名系统

## 一、项目概述

**LocalVidRen** 是一个 Windows 专属的本地短视频桌面应用程序，通过 AI 模型分析 + 语音识别技术，快速生成短视频内容总结，并自动将总结作为视频的新文件名。

### 核心特性
| 特性 | 说明 |
|------|------|
| 平台 | Windows 10/11 (64 位) 专属 |
| 目标文件 | 500M 以内的短视频 |
| 隐私安全 | 100% 数据不出电脑（本地模式） |
| 处理速度 | 秒级内容分析，短视频专属优化 |
| 自动化 | 文件夹监控 + 无人值守运行 |

---

## 二、核心功能需求

### 2.1 视频文件扫描与管理

- [x] 支持手动选择单个或多个文件夹进行扫描
- [x] 自动过滤大于 500M 的视频文件（可在设置中关闭）
- [x] 自动识别常见短视频格式：`.mp4, .mkv, .avi, .mov, .flv, .webm`
- [x] 生成可交互的视频列表，显示：原文件名、文件大小、时长、处理状态、新文件名预览
- [x] 支持手动勾选/取消勾选需要处理的视频
- [x] 支持按文件名、大小、时长、处理状态排序
- [x] 支持一键清空已处理视频记录

### 2.2 AI 模型内容分析（短视频专属优化）

**核心能力**：快速整合画面分析和语音识别结果，生成精准的视频内容总结

#### 分析模式（用户可自由切换）
| 模式 | 说明 |
|------|------|
| **本地模式** | 使用本地大模型进行推理，数据完全不出电脑 |
| **API 模式** | 调用第三方在线 API（OpenAI GPT-4V、通义千问、文心一言等） |

#### 分析深度（可配置）
| 模式 | 关键帧 | 适用场景 |
|------|--------|----------|
| **极速模式（默认）** | 3-5 个关键帧 + 完整语音识别 | 1 分钟以内短视频，速度最快 |
| **标准模式** | 5-10 个关键帧 + 完整语音识别 | 平衡速度和准确率 |

#### 分析流程
1. **音频分析**（优先）：语音识别作为主要依据
2. **画面分析**（补充）：识别人物、场景、事件、物体、文字
3. **内容总结**：生成 10-20 字的简洁、准确描述

#### 可配置项
- 总结长度和风格（简洁描述型、事件概括型、关键词型）
- 关键帧提取间隔
- 模型参数（温度、最大生成长度等）

### 2.3 自动重命名功能

- [x] 将 AI 生成的内容总结自动作为视频的新文件名
- [x] 支持自定义文件名模板：
  - `[内容总结].[扩展名]`（默认）
  - `[日期]_[内容总结].[扩展名]`
  - `[原文件名前缀]_[内容总结].[扩展名]`
- [x] 自动处理文件名冲突：添加 `(1)`, `(2)` 等后缀
- [x] 自动过滤 Windows 文件名非法字符
- [x] 支持撤销重命名操作（保留最近 50 条历史记录）
- [x] 提供"预览重命名"功能，确认无误后再执行

### 2.4 批量处理与自动化

- [x] 支持一键批量处理列表中所有未处理的视频
- [x] 支持设置同时处理的视频数量（并发数）
- [x] 支持处理完成后自动关机/休眠（可选）
- [x] 支持处理失败自动重试（可配置重试次数，默认 1 次）
- [x] 支持跳过已处理过的视频（通过文件哈希值识别）

### 2.5 文件夹实时监控

- [x] 支持添加多个监控文件夹
- [x] 实时检测监控文件夹中的新增视频文件
- [x] 自动过滤大于 500M 的新增文件
- [x] 新增视频自动加入处理队列并开始分析
- [x] 支持设置监控延迟（避免文件未完全复制完成就开始处理，默认 10 秒）
- [x] 支持监控子文件夹（可配置是否递归）

### 2.6 无人值守运行

- [x] 支持开机自动启动（通过 Windows 任务计划程序实现）
- [x] 支持最小化到 Windows 系统托盘运行
- [x] 支持后台静默处理，不影响用户正常使用电脑
- [x] 生成详细的运行日志，记录处理时间、结果、错误信息
- [x] 支持 Windows 系统通知处理完成和错误告警

## 三、技术实现要求

### 3.1 开发环境与依赖

| 组件 | 要求 |
|------|------|
| 操作系统 | Windows 10/11 (64 位) |
| 编程语言 | Python 3.11（推荐，兼容性最好） |
| 界面框架 | PyQt6（Windows 下原生体验好） |
| 视频处理 | FFmpeg（Windows 预编译版，轻量级配置） |
| 语音识别 | faster-whisper-base（默认，速度最快） |
| 多模态分析 | 可插拔接口设计 |
| 本地模型 | llama-cpp-python 加载 GGUF 格式多模态模型 |
| API 模型 | OpenAI 兼容接口 + 主流国内大模型 API |

### 3.2 AI 模型接口设计

**核心要求**：统一的抽象模型接口，所有模型实现相同方法

#### 内置支持的模型类型
| 类型 | 推荐模型 |
|------|----------|
| 本地多模态模型 | Qwen2-VL-2B-Instruct、MiniCPM-V-2_6 等轻量级 GGUF 模型 |
| 本地语音识别 | faster-whisper-base/small 系列 |
| 在线 API | OpenAI GPT-4V、通义千问 VL、文心一言、豆包等 |

#### 模型管理特性
- 支持用户自定义添加新模型，只需实现接口方法
- 模型配置独立存储，支持一键切换
- 本地模型自动检测可用硬件加速（CUDA/CPU）
- 支持保存多个模型配置，快速切换

### 3.3 短视频专属性能优化

| 优化项 | 说明 |
|--------|------|
| **语音识别优先** | 优先提取和处理音频，大幅提升处理速度 |
| **关键帧提取优化** | 固定间隔提取（每 2-3 秒 1 帧），无需复杂场景检测 |
| **模型预热** | 启动时预加载默认模型，避免首次处理延迟 |
| **内存/显存优化** | 处理完立即释放模型资源 |
| **临时文件优化** | 内存优先方式，减少磁盘 IO |
| **临时文件管理** | 存储在 `%TEMP%\LocalVidRen`，处理完成后立即清理 |

### 3.4 数据安全

- [x] 本地模式下所有处理过程完全在本地进行
- [x] API 模式下明确提示用户数据将上传到第三方服务器
- [x] 不收集任何用户信息
- [x] 临时文件处理完成后立即删除

## 八、开发注意事项

### 核心原则

1. **仅支持 Windows 系统**，不考虑 Linux/macOS 兼容性
2. **针对 500M 以内短视频优化**，追求极致处理速度
3. **语音识别是核心**，必须优先处理音频，再处理画面
4. **模型接口必须抽象**，易于添加新的模型类型
5. **本地模式和 API 模式清晰分离**，API 模式要有明确的数据上传提示
6. **资源管理严格**：处理完每个视频后必须立即释放所有资源
7. **文件名处理**：考虑 Windows 文件系统特殊字符限制
8. **文件夹监控**：避免重复处理和处理未完成复制的文件

### 技术要点

- 使用 PyQt6 作为界面框架，适配 Windows 深色/浅色主题
- 使用 `llama-cpp-python` 加载本地 GGUF 多模态模型
- 集成 `faster-whisper` 进行语音识别（支持 CPU/CUDA 加速）
- 集成 FFmpeg 进行视频处理
- 实现 OpenAI 兼容 API 接口和主流国内大模型 API 支持
- 所有代码需有详细中文注释
